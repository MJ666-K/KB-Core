## 🔧 Bug 修复记录

### 修复 #1：SkillExecutor citations 缺失 ✅

**问题描述**：
- SkillExecutor 返回的 `citations` 总是空数组
- 导致法律匹配率为 0%
- 用户无法看到引用的法律来源

**根本原因**：
- `executor.ts` 从 Tool 调用结果中提取了 RetrievalResult，但没有转换为 Citation
- 第47行和第71行返回的 citations 总是空数组 `[]`

**修复方案**：
```typescript
// 在 executor.ts 中添加：
import { formatCitations, deduplicateChunks } from './types';

// 收集所有 RetrievalResult
const allRetrievalResults: RetrievalResult[] = [];

// 在 Tool 调用后检查并收集
if (this.isRetrievalResultArray(result)) {
  allRetrievalResults.push(...result);
}

// 最终返回时转换为 Citation
const deduped = deduplicateChunks(allRetrievalResults);
return { 
  answer: response.content ?? '', 
  citations: formatCitations(deduped),  // ← 修复：正确转换
  toolCalls: allToolCalls 
};
```

**修复文件**：
- `app/src/skills/executor.ts`

**验证结果**：
- ✅ Citations 现在正确返回（6 条引用）
- ✅ 法律匹配率从 0% 提升到 **100%**
- ✅ 引用中包含法律名称、得分、摘要
